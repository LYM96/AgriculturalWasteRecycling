AgriculturalWasteRecycling

农业废弃物回收系统的设计与实现

一、系统概述

本系统是基于 SpringBoot + Vue 架构开发的农业废弃物回收系统，旨在实现农业废弃物回收全流程的数字化、规范化管理，衔接农户、回收企业等多方主体，提升废弃物回收效率，助力农业绿色发展。

二、开发环境与技术栈

- 后端语言：Java

- 后端框架：SpringBoot

- 前端技术：Vue

- 数据库：MySQL

- 数据库工具：Navicat

- 开发工具：IDEA（后端）、VS Code（前端）

三、核心用户角色

系统支持三端用户协同使用，分别为：管理员、农户、企业用户。

四、系统功能模块总览

本系统核心功能模块涵盖：废弃物发布、废弃物分类、废弃物查询、废弃物识别、评价反馈、客服与帮助、回收订单追踪、农户管理、回收企业管理、回收订单管理、废弃物信息统计、政策与资讯资源模块、系统管理、数据分析与报表。

五、各角色详细功能

5.1 农户

1. 废弃物查询：按废弃物种类分类展示，方便农户快速查找目标废弃物相关信息。

2. 政策与资讯资源模块：展示政府发布的农业废弃物回收相关政策文件、最新行业资讯及技术指导，提供废弃物分类与处理技术知识科普。

3. 废弃物发布：农户可自主发布废弃物的类型、数量、存放位置等关键信息，发起回收需求。

4. 回收订单追踪：实时查看订单状态，包括待接单、已接单、运输中、已完成等全流程状态。

5. 废弃物识别：集成第三方接口，支持农户上传废弃物图片等信息，自动识别废弃物类型。

6. 回收订单评价：订单完成后，农户可对回收服务进行好评或差评反馈，反馈信息同步至后台。

7. 回收订单结算：订单回收完成后，对应的环保积分自动发放至农户账号。

8. 意见反馈：农户可通过系统提交意见或建议，实现与平台的双向沟通。

5.2 企业管理员

1. 农户管理：查看所有农户的基础信息，支持对农户信息进行新增、删除、修改、查询（CRUD）操作。

2. 回收企业管理：维护企业自身基础信息，查询企业历史回收记录及相关统计数据。

3. 回收订单管理：对农户发布的废弃物需求自动生成回收订单，并实时追踪、更新订单状态。

4. 数据分析与报表：通过可视化图表展示废弃物回收量、回收种类等核心数据，生成统计报表，为政府部门决策提供数据支撑。



5.3 系统管理员

1. 用户管理：查看所有用户的基础信息，支持对农户信息进行新增、删除、修改、查询（CRUD）操作。

2. 回收订单管理：对农户发布的废弃物需求自动生成回收订单，并实时追踪、更新订单状态。

3. 数据分析与报表：通过可视化图表展示废弃物回收量、回收种类等核心数据，生成统计报表，为政府部门决策提供数据支撑。

4. 系统管理：配置系统角色权限，设置系统核心参数，保障系统安全稳定运行。
